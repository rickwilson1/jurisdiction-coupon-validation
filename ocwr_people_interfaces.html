Step 5 — Fulfillment splits by order type
Pickup path
  1. Customer arrives at assigned yard
  2. Presents email confirmation + ID
  3. <5 yds: QR scan → self-load with 5-gal buckets
  4. ≥5 yds: OCWR staff loads with equipment
  5. OCWR logs fulfillment in Greenery Log
Delivery path
  1. Greg receives auto-alert + SharePoint log entry
  2. Greg calls customer to schedule
  3. Hauler receives PDF manifest
  4. OCWR signs manifest at pickup
  5. Greg emails Ofelia Friday delivery schedule
What is automated vs. manual
Automated (Cloud Run + Power Automate)
  • Coupon code validation against jurisdiction
  • Routing logic (pickup vs. delivery, qty threshold)
  • Customer confirmation email dispatch
  • CC to Ofelia on all orders
  • Greg alert email for delivery orders
  • SharePoint delivery log write-back
  • Hauler manifest PDF generation
  • Friday delivery schedule compilation
  • Firestore audit log per order
  • Coupon file updates (5-min cache)
Remains manual
  • ID / address verification at yard gate
  • Material loading (self or staff)
  • Greenery Log entry by OCWR staff
  • Greg phone call to schedule delivery
  • Hauler manifest physical signature
  • Coupon exception support (Kendall/Emily)
  • Abuse investigation and escalation
  • Monthly reporting review (Agromin)
