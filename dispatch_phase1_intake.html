Agromin / OCWR — Order dispatch system · Phase 1 of 2

Phase 1 — Order Intake

Automated order routing, email dispatch, and Firestore logging. Delivery scheduling remains manual and is addressed in Phase 2.

Scope — what this phase covers
Phase 1 — Order Intake this document
Automated intake and notification. Delivery scheduling is manual.
  • Inbox watch detects program orders automatically
  • Routes each order: delivery vs. pickup, qty threshold
  • Sends correct email to customer automatically
  • Alerts Greg on every delivery order
  • Logs all orders to Firestore
  • Greg schedules deliveries manually — phone call, no tool
  • No visibility into pending vs. scheduled deliveries
  • Greenery Log remains a shared Excel spreadsheet
Phase 2 — Delivery Scheduling next
Adds scheduling tools, delivery visibility, and yard log.
  • Greg's delivery queue — all pending orders in one place
  • Greg logs confirmed date/time after calling customer
  • Customer gets "delivery scheduled" email automatically
  • Ofelia's live schedule view — no more asking Greg
  • Yard staff log fulfillments via browser form
  • Greenery Log replaces shared Excel
  • OCWR-formatted export for compliance reporting
Systems and people involved
Customer
Orange County resident
  • Places order at shop.agromin.com
  • Enters coupon code + address
  • Coupon validated at checkout
  • Receives confirmation email
  • Brings ID to yard or awaits delivery call
CIMcloud + Sage 100
Order & accounting system
  • Accepts or declines order at checkout
  • Sends confirmation email to sales@agromin.com
  • Syncs orders into Sage 100
  • No API access available to Agromin
  • Email inbox is the only readable output
Coupon Validator
Cloud Run · existing service
  • Called at checkout — before order is placed
  • Geocodes customer address via ArcGIS
  • Checks coupon vs. CDTFA tax district
  • Returns accepted or denied to CIMcloud
  • Validation complete before dispatch sees order
Power Automate
Inbox watch + order parser
  • Watches sales@agromin.com inbox
  • Triggers on emails with "Coupon Code:" field
  • Parses order fields from email HTML
  • POSTs structured JSON to dispatch service
  • Ignores all paid orders automatically
Dispatch Service
Cloud Run · new service
  • Receives parsed order from Power Automate
  • Routes: delivery vs. pickup, qty threshold
  • Sends correct email to customer auto
  • Alerts Greg on delivery orders auto
  • Logs order to Firestore store
Greg (Brian)
Delivery coordinator
  • Receives alert email per delivery order auto
  • Calls customer to agree on date/time manual
  • Assigns truck and coordinates haul manual
  • Requests PDF manifest for hauler
  • Ofelia CC'd on all customer emails auto

The Firestore log written here is the only structured, queryable record of the order outside of CIMcloud and Sage 100, neither of which have an available API.
Step 6 — Fulfillment splits by order type
Pickup path
  1. Customer receives email with yard address + hours auto
  2. Under 5 cu yds — self-load with 5-gallon buckets
  3. 5 cu yds or more — OCWR staff loads with equipment; truck/trailer required
  4. Customer shows email confirmation + photo ID at yard
  5. OCWR staff enters fulfillment in Greenery Log
Delivery path
  1. Customer receives delivery acknowledgment email auto
  2. Greg receives alert email with customer details auto
  3. Greg calls customer to agree on delivery date/time manual
  4. Greg assigns truck and coordinates haul manual
  5. Hauler carries PDF manifest; OCWR signs at pickup
Where data lives
CIMcloud / Sage 100
Order & accounting system
Source of record for all orders. No API access. Readable only via confirmation emails sent to sales@agromin.com.
Firestore — order_events
Google Cloud · NoSQL
Written by the dispatch service for every program order. The only place you can query dispatch history — routing decisions, customer details, qty — without reading emails.
GCS — agromin-coupon-data
Google Cloud Storage
Stores the active coupons.xlsx file. Used by the Coupon Validator at checkout. Updated by admin upload; refreshes within 5 minutes.
What is automated vs. manual
Automated in Phase 1
  • Inbox watch — no manual email scanning needed
  • Routing decision (delivery vs. pickup, qty threshold)
  • Customer confirmation email with correct instructions
  • Yard name, address, and hours populated from order
  • Ofelia CC'd on all order emails
  • Greg alert email on every delivery order
  • Firestore log written per order
  • PDF manifest generated on request
Remains manual — addressed in Phase 2
  • Greg calls customer to negotiate delivery slot
  • Truck assignment and haul logistics
  • Recording confirmed delivery date/time
  • ID and address verification at yard gate
  • Material loading (self or OCWR staff)
  • Greenery Log entry by OCWR staff
  • Hauler manifest physical signature
  • Exception handling and coupon support
